# Contributing to Loom

Thank you for your interest in contributing to Loom! We welcome contributions from the community and are grateful for any help you can provide.

## How to Contribute

### Reporting Bugs

If you find a bug, please open an issue with:

- A clear and descriptive title
- Steps to reproduce the issue
- Expected behavior vs. actual behavior
- Your environment (OS, Python version, Node.js version, browser)
- Any relevant logs or screenshots

### Suggesting Features

Feature requests are welcome. Please open an issue describing:

- The problem you're trying to solve
- Your proposed solution
- Any alternatives you've considered

### Submitting Pull Requests

1. Fork the repository and create your branch from `main`.
2. If you've added code, add or update tests as appropriate.
3. Ensure the test suite passes.
4. Make sure your code follows the existing style and conventions.
5. Write a clear commit message describing your changes.
6. Open a pull request with a description of what you changed and why.

## Development Setup

See [DEPLOYMENT.md](DEPLOYMENT.md) for instructions on setting up a local development environment.

## Code of Conduct

This project adheres to a code of conduct. By participating, you are expected to uphold respectful, inclusive, and constructive communication. Harassment, discrimination, and disruptive behavior will not be tolerated.

## Style Guidelines

- **Python (backend/agents):** Follow PEP 8 conventions.
- **TypeScript (frontend):** Follow the existing ESLint configuration.
- **Commits:** Use clear, concise commit messages. Prefer imperative mood (e.g., "Add feature" not "Added feature").
- **Documentation:** Update relevant documentation when your changes affect user-facing behavior.

## Getting Help

If you have questions about contributing, feel free to open a discussion or issue. We're happy to help you get started.
